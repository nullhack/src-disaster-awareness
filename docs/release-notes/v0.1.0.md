# src-disaster-report-v4 — v0.1.0

Initial release of the disaster-report ingest and reporting pipeline.
Released 2026-07-05. SemVer 0.x: no stability promise — any subsequent
0.x release may change the surface.

Tagged `v0.1.0` on `dev`. Local-only: no remote is configured and no
artefact left the repository.

## What it is

A pipeline that ingests geophysical and disease-outbreak signals from
public upstream sources, clusters them into sticky incidents, digests
news context through an AI model, and renders a two-part markdown brief
read from the operational store.

## Surface

### CLI

- `disaster-report ingest` — runs one batch through the pipeline;
  flags `--config`, `--secrets`, `--source`.
- `disaster-report report` — builds the read model and renders it to
  markdown; flags `--config`, `--secrets`.

### Python import surface

- `run_ingest` — orchestration entry; one batch → `IngestReport` counts.
- `build_report` — assembles the read model into a `ReportDocument`.
- `MarkdownRenderer` — renders a `ReportDocument` to a markdown string.
- `Warehouse` — SQLAlchemy-backed operational store; 12-method boundary.
- adapters — `USGSAdapter`, `GDACSAdapter`, `WHODiseaseOutbreakAdapter`,
  `DuckDuckGoNewsAdapter`.
- `OpenRouterDigester` — AI digester over LiteLLM; passthrough errors.
- `Clusterer` — sticky incident identity from the report/news graph.
- `Settings` — frozen TOML config + dotenv secrets, one canonical object.
- `SourceFetchError` — the per-adapter failure type the pipeline isolates.

## Behaviour summary

- **Source-adapter fetch with fail-isolation.** Each adapter raises
  `SourceFetchError` on a failed fetch; the pipeline catches per adapter
  so one broken source does not abort the batch. Every other exception
  (bad request, digester failure) propagates so the operator sees it.
- **AI digester passthrough.** Provider errors surface unchanged; the
  pipeline's isolation does not swallow them. A misconfigured key or
  model request is visible, not silently empty.
- **CIT sticky-identity.** The incident id is derived from the connected
  component of source reports and news items (union-find over the graph;
  canonical id is the lexicographically smallest member). Identity is
  stable across replays of the same upstream data. There is no
  `reclassify` or `redigest` surface — identity is not mutable.
- **Read-model + markdown brief split.** `build_report` reads incidents,
  timeline, and news into a `ReportDocument`; `MarkdownRenderer` splits
  the timeline into a geophysical section then a disease section by
  incident-id token, and HTML-escapes every interpolated summary so a
  hostile or odd AI summary cannot inject markdown or HTML.
- **TOML config + dotenv secrets.** `Settings` is the single frozen
  object the pipeline reads; the database url comes from TOML, the
  OpenRouter key from a dotenv file kept out of the workspace. The two
  never share a source. The api key is never logged or echoed.

## Known limitations and deferred work

These are carried forward as future work, not defects:

- `SourceReport.country` is hardcoded to `""` in the USGS adapter (the
  cassette carries no country field); GDACS country is structurally dirty
  and absorbed downstream — country normalisation is deferred.
- Cross-module `_as_dict` helper is duplicated between `usgs.py` and
  `who.py`; a shared private module is a future cleanup.
- GDACS `should_monitor` test asserts only `isinstance(_, bool)` — the
  alert-level filter is name-intent-correct (Orange/Red kept) but
  shape-weak; the test could be tightened.
- `DigestResult.selected_news` is a passthrough of `candidate_news` and
  `relevance_scores` is `{}` — the digester does not yet sub-select or
  score; field types permit the richer behaviour when a cycle requires
  it.
- Pipeline internals carry forward three micro-decisions:
  `_report_key` uses `"{source}:{source_id}"` (safe for current sources;
  revisitable if a source embeds `:`), the DDG search uses the `strict`
  key only (the `loose` key from `derive_keys` is unused), and
  `_prior_timeline_for` is URL-overlap based (correct for re-poll
  semantics; exercised only by the production digester).
- Reporting carries forward three items: `_DISEASE_TOKENS` is a seeded
  frozenset (an external source is needed if synonyms or multilingual
  terms are required), `build_report` makes N read round-trips
  (acceptable at current scale; flagged for a batch read-model design),
  and HTML-escaping covers the AI summary only — broader markdown-
  injection vectors (link href, image src) remain consumer
  responsibility.
- `Settings.load` carries a defensive trailing `or ""` on the db_url
  parse (TOML cannot produce `None` and `__init__` already normalises
  empty) — mildly redundant, kept defensively.
- CLI error paths echo the exception text to stderr (`cli.py` ingest and
  report handlers); litellm redacts the api key in practice, but the
  surface is developer-facing stderr, not stdout.
- Two e2e witnesses (`test_ai_not_called_for_filtered_or_no_news`,
  `test_all_already_ingested_drops_summary`) are redundant-weak — each
  passes on the merits as a consequence of correct wiring, and the
  stronger siblings in the same file enforce the rule.

## Architectural decision pending an ADR

The **fail-isolation location** is concretely encoded in code but not
yet captured in an Architecture Decision Record. The pipeline catches
`SourceFetchError` per adapter at `disaster_report/pipeline.py:69`
(orchestration owns the isolation policy; adapters know nothing about
it). The viable alternative — adapters self-isolate and return `[]` —
would distribute the policy across every future adapter. Both satisfy
the contract; the silence between them is the architectural decision.
This release ships with the pipeline-catch as the operative policy. A
`record-decision` pass is recommended; this release does not block on
it.
